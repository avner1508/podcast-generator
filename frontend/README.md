# Podcast Generator - Frontend

React/TypeScript SPA for creating, previewing, and publishing AI-generated podcasts.

## Tech Stack

- **React 19** + **TypeScript**
- **Vite** - build tool and dev server
- **Tailwind CSS 4** - styling
- **Axios** - HTTP client
- **React Router 7** - client-side routing

## Setup

```bash
cd frontend
npm install
```

## Running

```bash
npm run dev
```

Runs on `http://localhost:5173`. The Vite dev server proxies `/api` requests to `http://localhost:8000` (the backend), configured in `vite.config.ts`.

## Scripts

| Command | Description |
|---|---|
| `npm run dev` | Start dev server with hot reload |
| `npm run build` | Type-check + production build to `dist/` |
| `npm run preview` | Serve the production build locally |
| `npm run lint` | Run ESLint |

## User Flow

### 1. Create Podcast (`/`)

`HomePage` is the main form with these sections:

- **Document upload** (`DocumentUploader`) - drag-and-drop or browse for PDF/TXT/MD files. Files are uploaded immediately to the backend and text is extracted server-side. Multiple documents supported
- **AI provider** (`ProviderSelector`) - fetches available providers from `/api/providers` and displays them as selectable cards. Options: Gemini Full, Claude + Gemini TTS, Claude + ElevenLabs
- **Language** - dropdown with 12 languages (English, Hebrew, Spanish, French, German, Italian, Portuguese, Arabic, Russian, Japanese, Korean, Chinese)
- **Speakers** (`SpeakerConfig`) - 1-3 speakers with name, role (host/expert/interviewer), and gender (male/female). Default: 2 speakers
- **Episode title** - text input

Clicking "Generate Podcast" sends a `POST /api/podcasts/generate` and navigates to the generation page.

### 2. Generation Progress (`/generate/:jobId`)

`GenerationPage` shows real-time progress using the `useGenerationStatus` hook.

The hook uses **dual strategy** for reliability:
- **WebSocket** (`/api/ws/jobs/:jobId`) for instant updates
- **HTTP polling** (`GET /api/podcasts/jobs/:jobId` every 2 seconds) as fallback

The `GenerationProgress` component shows a progress bar and stage labels:
- Queued -> Parsing documents -> Generating script -> Generating audio -> Assembling -> Complete

On completion, auto-redirects to the preview page after 1.5 seconds.

### 3. Preview (`/preview/:podcastId`)

`PreviewPage` loads the podcast details and shows:
- Title, provider, speaker count, duration
- Summary (generated by the AI)
- **Audio player** (`AudioPlayer`) - custom player with play/pause, seek bar, time display, and playback speed control (0.5x to 2x)
- **Publish button** (`PublishButton`) - opens a form to upload to Podbean with optional description and draft/publish toggle

### 4. History (`/history`)

`HistoryPage` lists all generated podcasts with status badges:
- **Published** (green) - uploaded to Podbean
- **Ready** (blue) - audio generated, not yet published
- **Processing** (gray) - still generating

Each item links to its preview page.

## API Client

`api/client.ts` wraps all backend calls using Axios with base URL `/api`:

| Function | Endpoint | Description |
|---|---|---|
| `uploadDocument(file)` | `POST /documents/upload` | Upload file via FormData |
| `listDocuments()` | `GET /documents` | List uploaded documents |
| `deleteDocument(id)` | `DELETE /documents/:id` | Remove a document |
| `listProviders()` | `GET /providers` | Get available AI providers |
| `generatePodcast(request)` | `POST /podcasts/generate` | Start generation |
| `getJobStatus(jobId)` | `GET /podcasts/jobs/:jobId` | Poll job progress |
| `listPodcasts()` | `GET /podcasts` | List all podcasts |
| `getPodcast(id)` | `GET /podcasts/:id` | Get podcast details |
| `getAudioUrl(podcastId)` | (constructs URL) | Returns audio stream URL |
| `publishPodcast(id, desc, draft)` | `POST /podcasts/:id/publish` | Publish to Podbean |

## Type Definitions

All shared types are in `types/index.ts`:

- `Speaker` - name, role, gender, voice_id
- `DocumentInfo` - id, filename, content_type, text_preview, created_at
- `ProviderInfo` - id, name, description, script_engine, tts_engine
- `JobStatus` - job_id, podcast_id, stage, progress, error
- `PodcastInfo` - id, title, summary, provider_id, num_speakers, duration, audio_path, podbean_episode_id, created_at
- `GenerateRequest` - document_ids, provider_id, speakers, title, language
- `GenerateResponse` - job_id, podcast_id
