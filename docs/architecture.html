Podcast Generator

Document-to-podcast platform with AI-powered script generation, multi-speaker TTS, and Podbean publishing

Python / FastAPI React / TypeScript Gemini 2.5 Claude ElevenLabs Podbean

System Overview

Frontend (React + Vite)

  • HomePage Upload docs, configure speakers, select provider
  • GenerationPage Real-time progress via WebSocket / polling
  • PreviewPage Audio player, summary, publish to Podbean
  • HistoryPage List of all generated podcasts

Backend (FastAPI + SQLite)

  • POST /api/documents/upload PDF/TXT/MD parsing
  • POST /api/podcasts/generate Async pipeline (HTTP 202)
  • WS /api/ws/jobs/{id} Real-time job status
  • GET /api/podcasts/{id}/audio Stream MP3
  • POST /api/podcasts/{id}/publish Podbean upload

Storage

  • SQLite Podcast metadata, documents, job state
  • ./uploads/ Uploaded document files
  • ./output/ Generated MP3 audio files

External APIs

  • Gemini 2.5 Pro Script generation
  • Gemini 2.5 Flash TTS Multi-speaker voice synthesis
  • Claude (Anthropic) Script generation (alt provider)
  • ElevenLabs Premium voice synthesis (alt provider)
  • Podbean API OAuth2, upload, episode creation

Generation Pipeline

1
10–20%
Parse Documents
Extract text from uploaded PDFs (PyMuPDF), TXT, and MD files
Concatenate all documents with separators
2
25–50%
Generate Script
LLM creates a natural podcast dialogue between configured speakers
Output: structured JSON — [{speaker, text}, ...] + summary (≤300 chars)
3
55–80%
Generate Audio
TTS provider converts script to multi-speaker audio
Gemini: native 2-voice, auto-chunks at ~580s, PCM→MP3  |  ElevenLabs: per-line, async parallel
4
85%
Assemble Audio
Single segment → write directly  |  Multiple → concatenate
Output: MP3 file ready for playback and Podbean upload
✓
100%
Complete
Audio path + duration saved to DB. Frontend redirects to preview page.
